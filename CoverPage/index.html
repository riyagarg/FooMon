Metrics @ real time

Foomon provides an out of the box support for industry grade metrics monitoring system that works with servers, applications or even custom data that you can push for a short while.

What does it cover ?

  • Applications : Microservices or services, Java , Ruby or node based applicaitions
  • Servers : Bare metals, vms or AWS hosts, this data ranges from cpu usage , entropy disk space, mysql read writes , kafka latencies and a huge number of other metrics
  • Custom cron jobs / On demand data : You can also pipe in data from custom shell scripts into the system to visualise it at a later point in time.

    • Alerting on slack and telegram

    Replaces Logentries

    Apart from numeric metrices, foomon also hosts an elasticsearch instance and a kibana dashboard where logs from various applications are steamlined and and is indexed for querying. The logs are available as soon as you produce them , making it super easy to query them in case of issues or when you want to make reports out of them in terms of charts

    The whole thing is supported by filebeat, which also makes sure you are not overloading the elasticsearch intance
